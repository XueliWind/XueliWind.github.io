一、 按钮和动态菜单面板

1.UI按钮

按钮可帮助用户触发即时操作。 它是 MRTK 提供的基础组件之一。 根据具体情况，可以使用各种类型的按钮预制。 按钮基于可交互脚本组件，在不同的交互式图面上提供平滑的 UI 控件。

2.UI菜单

菜单是一个 UX 控件，它提供一组按钮或不同的 UI 组件。 它可跨用户框架浮动，随时轻松可用。 由于它与人员松散耦合，因此它不再影响用户与目标内容的交互。 用户可以使用“固定”按钮来锁定/解锁菜单。 可在所选位置抓取菜单，并重定位到所选位置。

四、使用场景和工具提示上的文本为对象提供说明

1.Textmesh Pro

一些 MRTK 资产和预制需要 TextMesh Pro。 TextMesh Pro 是 Unity 的文本组件。 TextMesh Pro 通过直接更改材料属性来帮助动态更改外观。 TextMesh Pro 使用自定义着色器和高级文本渲染技术来提高视觉质量。 必须通过导入 TMP 基本资源来将 TextMesh Pro 添加到项目中。

2.工具提示

工具提示是与一些其他操作或对象相关的简短说明。 工具提示用于在仔细检查时提供提示或额外的数据。

五、使用边界控件操作三维对象

1.若要使用跟踪手操作和抓取对象，这些对象必须附加以下组件：

  • 碰撞体：碰撞体组件概述了用于身体碰撞的 GameObject 的形式。 不可见的碰撞体不需要与 GameObject 的网格具有完全相同的形式。 在游戏中，网格的粗略近似通常更有效且不可区分。
  • 对象操控器(脚本)：ObjectManipulator 脚本能够让对象变得可移动、可缩放和可旋转，这些操作可通过一只或两只手来实现。 对象操控器可以配置为操作项目将如何响应大量输入。
  • NearInteractionGrabbable：NearInteractionGrabbable 组件可帮助你抓取其跟踪手附近的对象。

2.边界控制

BoundsControl 提供了一种简单的功能，用于在混合现实中对对象进行重新建模。 边界控件将在整个全息影像中显示一个容器，以表明可以与之交互。 框的角和边缘的控点将允许缩放、旋转或转换对象。 边界控件还会对输入做出反应。 例如，在 HoloLens 2 上，边界控件响应手指邻近，提供可见的备注以帮助了解对象中的空间。 所有交互和视觉对象都可以轻松进行自定义。

二、使用集合组织和布局对象

​ 在“集合”中排列对象是混合现实工具包提供的另一项实用功能。 对象集合是一个脚本，用于帮助以预定义的 3 维形状为对象数组布局。 它支持各种表面样式，包括平面、圆柱体、球体和径向。 因为它支持 Unity 中的任何对象，所以它可以用来对 2D 和 3D 对象进行布局。

1.网格对象集合

​ 可以在一组子对象上使用此类型的集合，以使用一些布局参数来组织它们。 网格对象收集可帮助快速创建控制面板或一组 Prefab 或对象。

2.散点对象集合

​ 此类型的集合只是在半径内随机布局的一组子对象。

3.图块网格对象集合

​ 可以利用这种类型的集合在基本网格布局中放置大小一致的游戏对象。

三、使用求解器创建智能对象放置

1.解释：

​ 如果最重要的选项出现在周围的菜单中，而不是固定在某个位置。混合现实工具包提供了此功能，称为求解器。 求解器使用预定义的算法计算对象的位置和方向。 求解器提供了各种行为，用于将对象附加到其他对象或系统。 例如，求解器可以是每次移动时显示的跟随对象（基于相机）。

​ 方向指示器是另一种类型的求解器。 如果想帮助跟随组件将其定向到空间中对象的所需方向，则可以使用该求解器。

​ 另一个需要了解的是点击以放置组件，它可以帮助将对象放置在表面上。 如果要将对象放置在空间网格上，此组件非常有用。

​ MRTK 的求解器位于 MRTK SDK 文件夹中。

2.练习使用方向指示器求解器

1.添加的组件：
  • DirectionalIndicator：确定作为方向指示器的对象的位置和方向。 从 SolverHandler 跟踪目标的参照点来看，此指示器将面向提供的 DirectionalTarget。
2.进入Game模式，按住鼠标右键旋转方向，当视线从RoverExplorer对象移开时，Indicator对象将出现并指向RoverExplorer对象。

3.练习使用点击以放置求解器

1.点击以放置

​ “点击以放置”是一个远距离交互组件，用于将游戏对象放置在表面上。 此组件适用于将对象放置在空间网格上。 “点击以放置”使用两个单击和头部运动的组合来放置对象。 一个单击用于开始放置，头部运动用于控制对象的位置，另一个单击用于将对象放置在场景中。

一、混合现实工具包（MRTK）

简介：
  • 混合现实工具包 (MRTK) 是用于构建虚拟现实 (VR) 和增强现实 (AR) 的混合现实体验的跨平台工具包。能够凭此创建一种混合现实体验，在其中用户能够移动、旋转和缩放全息对象。

二、为 Windows Mixed Reality 配置 Unity

  • 首先在Unity Hub中创建3D Core项目，并取名为”MRTK Tutorial“。

  • 切换生成平台，切换成”Universal Windows Platform“。

三、导入和配置资源

  • 导入 MRTK Unity 基础包。从 Microsoft 下载中心下载混合现实功能工具的最新版本。

四、完成为Unity配置项目

五、交互模型

​ 有三种主要的交互模型适合大多数混合现实体验。

  • 手部和运动控制器模型

    手部和运动控制器模型要求用户使用一只或两只手与全息影像交互。

    三种手部和运动控制器的形态：

  1. 用手直接操作
  2. 用手指向并提交
  3. 运动控制器

  • 免手动模型

​ 免手动模型使用户无需用手即可以与全息内容进行交互。 但是可以使用语音输入或“凝视并停留”。

  • 凝视并提交

​ 在与无法触及的全息内容交互时，最好使用“凝视并提交”。 用户凝视一个对象或 UI 元素，然后使用辅助输入单击它（“提交”到它）。 提交方法包括语音命令、按下按钮或手势。
